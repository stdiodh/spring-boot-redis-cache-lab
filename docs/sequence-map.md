# 시퀀스 맵

## 이 레포 안의 성장 흐름

### 07. 캐시와 Redis

- 게시글 단건 조회 앞에 Redis 캐시를 붙입니다.
- 먼저 캐시를 보고, 없으면 DB를 보고, 다시 캐시에 저장합니다.
- cache hit와 miss를 비교합니다.
- TTL로 캐시가 영구 저장소가 아니라는 점을 확인합니다.

## 왜 이 레포를 따로 분리했는가

`spring-boot-db-access-lab`은 저장, 요청 검증, 인증, 테스트까지의 기본기 흐름을 담고 있습니다.
반면 이번 주제는 성능 보조와 캐시라는 새로운 학습 도메인에 더 가깝습니다.

그래서 07부터는 새 토픽 레포로 분리해 조회 최적화와 캐시 문제를 별도로 다룹니다.
