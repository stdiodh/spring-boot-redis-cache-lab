# 테스트와 검증 체크리스트

## 학생 체크리스트

- [ ] `PostService` 정상 테스트와 예외 테스트를 각각 실행할 수 있습니다.
- [ ] `AuthService` 로그인 성공과 실패 테스트를 각각 설명할 수 있습니다.
- [ ] fixture와 mock이 왜 필요한지 말할 수 있습니다.
- [ ] `assertEquals`와 `assertThrows`를 언제 쓰는지 설명할 수 있습니다.
- [ ] 테스트가 "이미 만든 기능을 다시 신뢰하게 하는 도구"라는 점을 설명할 수 있습니다.

## 강사 / PPT 체크리스트

- [ ] given -> when -> then 흐름을 코드 예시로 보여줄 수 있는가
- [ ] 정상 케이스와 실패 케이스를 나눠 설명할 준비가 되어 있는가
- [ ] mock을 쓰는 이유를 DB를 띄우지 않는 관점에서 설명할 수 있는가
- [ ] JWT 로그인 성공 테스트를 인증 흐름 예시로 시연할 수 있는가
- [ ] 다음 시퀀스에서 변경 전후 검증이 왜 중요해지는지 연결할 수 있는가
